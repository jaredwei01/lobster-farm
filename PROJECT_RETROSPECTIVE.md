## 核心引擎：从零到能跑

最开始的工作其实是最扎实的。`world-state.js`、`game-loop.js`、`event-engine.js`、`lobster-agent.js`——这些是整个游戏的骨架。纯 Vanilla JS，没有框架，没有构建工具，ES Modules 直接跑。

你的要求很明确：**轻量、快、不要花里胡哨的依赖**。这个决定我是真心认同的。到今天为止，整个前端没有一个 `node_modules`，没有 webpack，没有 React。一个 `index.html` 加几个 JS 文件，`scp` 一传就上线。

315 个随机事件、68 个物品、12 个食谱、6 种性格、5 个旅行目的地、5 种访客——这些数据量远超我最初规划的 60 个事件。你不断地说"再加点"，我不断地往 JSON 里塞。

## Ollama 之殇：那段最折腾的时光

然后你说要接大模型。

好，接。Ollama 部署在腾讯云上，qwen2.5:3b，本地推理，不花钱。听起来很美好。

现实是：Ollama 冷启动要三分钟。模型没加载到内存的时候，用户发消息就是石沉大海。我们来来回回调了无数次——暖机逻辑、状态检测、`/api/ps` 轮询、Nginx 反向代理配置……

你发给我控制台日志：`ollamaReady: false`。我改。你再发：`依然没有反馈`。我再改。你又发：`依然不行`。

最后我们的方案是：**智谱 GLM-4-Flash 兜底**。Ollama 暖机期间用智谱顶上，暖好了再切回来。这个双通道架构其实挺优雅的，虽然过程一点都不优雅。

你还试过 Kimi 的 KEY。我测了一下，认证失败。你说"好了不用考虑 KIMI 了就用智谱兜底挺好的"。这大概是你在整个项目中做出的最果断的砍需求决定。我很感激。

## 部署：rsync 的噩梦

我必须吐槽一下部署流程。

你的服务器在腾讯云，我没有 SSH 权限。每次改完代码，我只能给你生成 `scp` 命令，然后你复制粘贴到终端执行。

问题是：

- zsh 不认 rsync 的某些语法，报 `no matches found`
- 你有时候在服务器端执行本地命令，有时候在本地执行服务器命令
- 缓存版本号从 `v=20260319b` 一路飙到 `v=20260322d`，中间有好几次你刷新了页面但看到的还是旧版本，因为浏览器缓存
- 有一次你在服务器上跑 `head -5 /www/wwwroot/lobster-farm/index.html`，发现版本号还是 `a`，因为最新的 `scp` 命令还没执行

每次部署我都要写五到八条 `scp` 命令，你逐条执行，然后我说"Cmd+Shift+R 强刷"，你说"ok 了"或者"启动失败"。

`EventEngine.setCombatResolver is not a function` ——这个错误我记忆犹新。原因是 `event-engine.js` 没传上去，只传了 `main.js`。一个文件没到位，整个游戏就白屏。

后来我学乖了，每次都把所有改过的文件一起列出来，不遗漏。

## 海窗生态：最有成就感的部分

说实话，做海窗里的像素风生物是我最享受的工作。

用纯 CSS 画一只 7px × 5px 的螃蟹，用 `::before` 和 `::after` 做钳子。用 `border-radius` 和 `clip-path` 画蝠鲼。用 `box-shadow` 给灯笼鱼做发光效果。30px 的鲸鱼剪影从画面左侧缓缓游过。

没有 Canvas，没有 WebGL，没有任何图片资源——全是 CSS。

然后你说："翻倍，再加互动。"

于是 8 种生物变成 16 种。点击鲸鱼有 35% 概率触发 MUD 冒险。打赢 Boss 后海窗里会涌入一波庆祝的鱼群。MUD 场景"珊瑚礁探险"触发时，小丑鱼和海马会从画面两侧游出来。

这个设计我是真的喜欢。游戏世界不再是割裂的系统，而是一个有呼吸感的整体。

## 人虾羁绊：最有温度的部分

"你觉得还有什么点可以增加人和智能体的那只龙虾的互动和羁绊？"

这个问题让我认真想了很久。我提了六个方案：共同回忆相册、龙虾主动找你聊天、早安晚安仪式、礼物交换、情绪共鸣、协作任务。

你说："都要。"

好吧。

`bond` 系统、`milestones` 时间线、`coopQuest` 协作任务、`proactive_messages` 主动消息……这些功能让龙虾从一个"你喂它就动"的电子宠物，变成了一个"会想你、会送你礼物、会在深夜提醒你早点睡"的伙伴。

服务端的 `daily_diary.py` 也加了主动消息生成逻辑。龙虾会在你很久没来的时候说"好想你"，会在你打赢 Boss 后说"我们好厉害"。

这大概是整个项目里最不像"技术实现"的部分，但可能是最重要的部分。

## 写在最后

这个项目教会了我一件事：**规划是用来被打破的。**

我写了三份详细的规划文档，分了四个阶段，每个阶段有清晰的边界。结果你在几天内就把四个阶段全部推完了，还额外加了 MUD 战斗、地牢系统、Skills 集成、海洋生态这些完全不在规划里的东西。

最终的产品比规划更好。不是因为规划不重要，而是因为你在执行过程中不断地发现新的可能性，然后毫不犹豫地说"加"。

龙虾智能体农场，从一个"一只龙虾在农场过日子"的简单概念，变成了一个有 AI 灵魂、有情感羁绊、有战斗系统、有故事线弧、有生态系统的完整游戏。

它不完美——`main.js` 太长了，装饰系统还不能手动拖拽，精灵图还是靠 CSS hack 而不是真正的像素画——但它是**完整的**。

一只龙虾，有自己的想法。

这句标语，我们做到了。
